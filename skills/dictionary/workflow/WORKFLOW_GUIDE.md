# 单字技能组合指南 (Workflow Definition)

本指南旨在说明如何将多个“单字技能”组合成一个连贯的工作流。

## 核心概念

1.  **单字技能 (Character Skill)**: 最小的功能单元，专注于完成单一、原子的任务。例如：“读”、“写”、“查”、“算”。
2.  **工作流 (Workflow)**: 单字技能的有序排列，通过数据流连接。

## 组合原则

1.  **原子性**: 每个步骤必须是一个独立的单字技能。
2.  **数据流**: 前一个字的输出必须能够作为后一个字的输入。
3.  **明确性**: 每个步骤的意图必须清晰，避免歧义。

## 组合示例

假设我们有两个单字技能：
- **【读】**: 读取指定路径的文件内容。
- **【写】**: 将内容写入指定路径。

我们可以组合成一个“复制”工作流：
1.  调用 **【读】** 技能，读取 `source.txt`。
2.  获取 **【读】** 的输出内容。
3.  调用 **【写】** 技能，将内容写入 `destination.txt`。

## 定义语法 (伪代码)

```python
workflow copy_file(source, destination):
    content = 读(source)
    写(destination, content)
```

## 执行逻辑

在 Trae 中，工作流通常体现为一个 Python 脚本或一个复合技能，它按顺序调用其他技能。
