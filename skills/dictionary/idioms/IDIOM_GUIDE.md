# 成语工作流指南 (Idioms - Special Workflows)

本指南定义了“成语”在仓颉造字体系中的特殊地位。成语不仅仅是单字的简单组合，而是封装了特定业务逻辑、最佳实践或复杂模式的“高级工作流”。

## 什么是成语？

成语 (Idiom) 是经过验证的、具有特定语义的单字组合模式。它通常对应一个复杂的任务，如“循序渐进”、“触类旁通”。

### 特征
1.  **语义封装**: 成语的含义大于其组成单字之和。
2.  **固定模式**: 内部的单字调用顺序和参数传递方式是固定的。
3.  **直接调用**: 用户可以直接调用成语，而无需关心内部的单字细节。

## 示例

### 成语：【循序渐进】 (Sequential Thinking)
- **组成**: [思] -> [拆] -> [行] -> [验]
- **含义**: 对复杂任务进行深度思考，拆解步骤，逐步执行并验证。
- **用法**: 当用户通过 `dictionary` 技能查询“如何处理复杂任务”时，可以直接返回【循序渐进】成语的定义。

## 定义方式

成语通常被定义为一个独立的 Python 脚本或复合技能，存放在 `.trae/skills/idioms/` 目录下（假设存在该目录，或在 `dictionary/idioms` 中定义）。

```python
# 伪代码
def idiom_sequential_thinking(task):
    plan = 思(task)
    steps = 拆(plan)
    for step in steps:
        result = 行(step)
        验(result)
```
